# PirateIsles — Assets

Все ассеты игры хранятся здесь. Ниже описание структуры и соглашения по именованию.

---

## Структура папок

```
assets/
├── sea/                    # Море: волны, пена, рябь
├── ui/
│   ├── buttons/            # Фоны и состояния кнопок UI
│   └── icons/              # Иконки ресурсов, статусов, действий
├── sprites/
│   ├── ships/              # Корабли (игрок, AI-рейдеры, PvP-ракеты)
│   ├── islands/            # Острова (базовый, повреждённый, разрушенный)
│   ├── buildings/          # Постройки (таверна, dok, пушки, уровни)
│   └── effects/            # Эффекты (огонь, дым, щит, взрыв, обломки)
├── fonts/                  # Шрифты (если подключаем локально)
└── audio/                  # Звуки и музыка (SFX / BGM)
```

---

## sea/

Текстуры и тайлы для поверхности воды.

| Файл | Описание |
|------|----------|
| `water_tile.png` | Базовый тайл воды (256×256 или 512×512, seamless) |
| `foam_tile.png` | Пена/гребни волн (seamless, с прозрачностью) |
| `wave_normal.png` | Normal map для волн (если будет WebGL) |
| `caustics_*.png` | Каустика (опционально, анимация: caustics_00..15.png) |

**Формат:** PNG, степень детализации: 256×256 px на тайл.
Тайлы должны быть seamless (бесшовные).

## sprites/ships/

Спрайты кораблей. Каждый корабль — отдельный PNG, вид сверху (top-down).
Если нужна анимация — нарезать в спрайт-лист (spritesheet).

| Файл | Описание |
|------|----------|
| `ship_player.png` | Корабль игрока (вид сверху, нос → вправо) |
| `ship_ai_raider.png` | AI-рейдер (орбитальный вокруг острова) |
| `ship_pvp_dart.png` | PvP-снаряд / летящая лодка атаки |
| `ship_enemy.png` | Упрощённый вид чужого корабля на world-view |
| `ship_wreck.png` | Обломок (после потери лодки) |

**Формат:** PNG с прозрачностью. Размер: 32×32 или 48×48 px.
Ориентация: нос смотрит вправо (+X), поворот в коде через `ctx.rotate()`.

## Соглашения

- **Именование:** `snake_case`, без пробелов, без кириллицы в именах файлов.
- **Прозрачность:** всегда PNG с alpha там, где нужен прозрачный фон.
- **Масштаб:** дизайн под 1× (pixel-perfect), масштабирование — в коде через canvas scale.
- **Спрайт-листы:** горизонтальная нарезка, все кадры одного размера.
- **Placeholder-графика** в коде (ctx.fillRect и т.д.) остаётся до тех пор, пока не добавлен реальный ассет — просто подключить через `new Image()`.
